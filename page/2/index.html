共享代理.md

Posted on 2016-10-07

在笔记本上运行着lantern，监听的端口是127.0.0.1:8787，127.0.0.1是本机的IP地址，就意味着只有本机的程序可以访问，如果手机等其它设备，就需要将端口转发为0.0.0.0:8787，但8787已经被占用，就用0.0.0.0:40404^1
需要用到的工具rined，有windows版和linux版，放在了Dropbox/Temp下，配置文件config.txt内容如下 0.0.0.0 40404 127.0.0.1 8787 作用就是任意地址的程序都可以访问40404端口，该端口又将数据转发至127.0.0.1:8787
然后在手机上就可以设置代理为电脑的IP地址，端口40404

除了把端口号设置为6位，感觉没有放什么错，但一直没有成功。晚上又试了次，终于成功了。失败，其中可能有我无法想到的原因，但如果不多次尝试，那么就肯定失败了。

Hexo 博客

Posted on 2016-10-06

以前的博客是多年前用Jeklly生成的，最近Github提示我在用的markdown解析器将不再支持，按找提示修改了，但用浏览器打开的时候md格式无法解析了，以前我就没有把jeklly搞懂，现在怕麻烦就懒得再去研究，最近看到的博客有一个关键词“hexo“，xiaolai.li写得人人都是工程师教程也以hexo为例子，尝试了下，原来很简单，换个主题就是复制粘贴。

Basic usage：

  1. 新建一篇博客
    hexo new <title>

  2. 生成博客（Generates static files）
    hexo generate
    写了一篇文章后，一开始没有理解这条命令的作用，就使用‘hexo deploy’，git 提示没有文件可以提交，好奇怪啊，不是命名在文件系统中多了一个文件了吗，可能是git的机制是不generate，就认为是不存在的对象。
    这条命令还有一个参数 -d 在成长文件的时候就可以发步到网上，加上这个参数，其实就相当于hexo generate & hexo deploy,不过写的文章应该仔细检查，为了缓冲，还是用：

  3. hexo deploy 发布
